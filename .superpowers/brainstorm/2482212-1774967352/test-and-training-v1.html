测试流程 & 分阶段训练流程

整体测试覆盖：模块测试 + 集成测试 + 端到端 + 端侧模拟验证

一、整体测试流程（Test Pipeline）

阶段1：模块级单元测试（Module Tests）

  • 检测器：mAP@0.5、mAP@0.5:0.95、Recall（针对小目标/遮挡人体单独评估）。输入：私有跌倒数据集的人体框标注
  • 跟踪器：IDF1、MOTA、ID Switch 次数。输入：检测器输出的伪标注序列
  • 关键点：PCK@0.2、OKS-AP（若用 COCO 预训练则先在 COCO val 上跑；若补充标注则在私有跌倒数据上测）。NPU/量化后：定点输出与浮点输出的平均 L2 误差
  • 分类器：Accuracy、Precision、Recall、F1、混淆矩阵（跌倒 vs 非跌倒：弯腰、蹲下、坐下等难例）。输入：跌倒数据集的 clip 段
  • 规则引擎：规则触发率、误触发率（在非跌倒动作上的触发比例）。通过 gtest/pytest 做逻辑覆盖

阶段2：集成测试（Integration Tests）

  • 检测+跟踪：测试抽帧下跟踪的连贯性（ID 漂移、遮挡恢复）
  • 检测+跟踪+关键点：关键点对检测框偏移、尺度变化的鲁棒性
  • 规则+分类器：规则触发后，分类器能否正确纠正规则误触发（如蹲下被规则误判为跌倒）
  • 时序稳定性：同一段跌倒视频的多次运行结果方差（蒙特卡洛稳定性）

阶段3：端到端 Inference Pipeline 测试

  • 视频级测试：给定 1080p 视频，输出完整结果（带可视化：框/跟踪 ID/关键点/跌倒状态）。对比 GT 标注计算：事件级 TP/FP/FN、漏报率、误报率/小时
  • 实时性能测试：在 x86（GPU）上测 FPS、延迟 P50/P99；记录不同抽帧率下的端到端延迟
  • 鲁棒性测试：光照变化、遮挡物（家具/其他人）、不同服装颜色、侧卧 vs 仰卧 vs 趴下

阶段4：端侧模拟部署验证

  • 模型转换链路：PyTorch → ONNX → 海思 wk/NNIE 格式（或 nnief/ruyi 工具链）。验证每一层输出与浮点一致（cosine similarity > 0.99）
  • 量化验证：INT8 量化前后精度对比（检测 mAP 掉点、关键点 PCK 掉点、分类器 accuracy 掉点）。量化校准数据集从训练集里抽取 100~200 张
  • ARM 模拟/真机验证：若无法直接上海思，先用 ARM CPU（如 RK/树莓派）跑 ONNX/NCNN 做功能验证；再在交叉编译环境模拟后处理逻辑
  • 资源占用评估：峰值 DDR、模型加载 flash 大小、每帧前向耗时预估（用 FLOPs / 0.5T 做理论估算 + 实际子模块 benchmark）

二、分阶段训练流程（Training Pipeline）

核心原则：检测器和关键点先在公开数据集上做并行预训练；跟踪器基于检测结果做参数调优；分类器串行在（检测+关键点+私有数据）之后训练；最后端到端串联并冻结/corrective fine-tune。

阶段1：检测器

数据集：MS COCO person + 私有跌倒人体框

步骤：

  1. COCO 预训练 80 epochs（人体类权重做初始化）
  2. 私有数据 fine-tune：调整 anchors（如果有）、数据增强（Mosaic、Mixup 可选，但端侧建议关掉重增强）、输入 832×448
  3. 量化感知训练（QAT）或 PTQ 准备：导出 ONNX

验证：mAP/Recall → 通过

阶段2：17关键点

数据集：MS COCO keypoints（或 CrowdPose）

步骤：

  1. COCO 预训练：MobileNetV2/ShuffleNet backbone + SimHR / SimpleBaseline head（轻量版）
  2. 输入 256×192 或 128×96（权衡精度与 DDR）
  3. 若后续业务需要提升：补充跌倒场景的关键点标注做 domain adaptation（阶段4之后可选）

验证：PCK@0.2 / OKS-AP → 通过

阶段3：跟踪器

方式：不训神经网络，纯调参

  1. 在阶段1检测器输出的私有视频序列上运行 ByteTrack-lite
  2. 调优：det_thresh、track_thresh、match_thresh、max_age、min_hits
  3. 评估 MOTA/IDF1，确认 ID Switch 在可接受范围

阶段4：姿态分类器

数据集：私有跌倒数据集（跌倒 / 非跌倒 clip）

  1. 用阶段1检测器 + 阶段2关键点在训练视频上提取“真值特征”（ROI 图 + 关键点 + 运动轨迹）
  2. 构建融合分类器，训练 100~200 epochs，加 Dropout 防止过拟合
  3. 难例挖掘（混淆矩阵分析），补充非跌倒难例数据

验证：Accuracy/F1 → 通过

阶段5：端到端串联与整体微调

  1. 冻结检测器、关键点、分类器权重，在私有验证集上跑整条 pipeline
  2. 若存在明显级联误差（如检测框抖动导致关键点不稳定、分类器对检测误差敏感），可做 corrective fine-tune：在端到端数据上微调检测器/分类器，但端侧小网络调整幅度要小
  3. 规则引擎阈值（T_trigger / T_alarm / 各权重）在验证集上做网格搜索
  4. 输出最终部署模型 + 配置文件

训练时模型与数据接口约定

  • 统一数据格式：COCO JSON（人体框）+ 自定义 CSV/JSON（跌倒事件时间戳/标签）
  • 中间特征缓存：为阶段4分类器训练，自动缓存检测框、关键点、运动特征到 pickle/NPY，避免每次重跑前向
  • 可复现：每个阶段固定随机种子、保存训练配置 yaml、保留最佳 checkpoint
