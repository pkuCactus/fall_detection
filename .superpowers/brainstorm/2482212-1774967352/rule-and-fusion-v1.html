规则引擎 + 分类器融合设计

规则触发 → 二阶段验证 → 融合决策

一、规则引擎（快速筛选 / 低算力持续运行）

输入：连续 N 帧的 17 关键点 + Track 历史轨迹

输出：S_rule ∈ [0, 1]（疑似跌倒得分）。当 S_rule ≥ T_trigger 时，启动分类器二阶段验证。

A. 高度压缩判定

  • H_ratio = (head_y - ankle_y) / bbox_h。正常站立时 H_ratio ≈ 0.85~0.95，跌倒时会大幅压缩
  • 多点贴地：统计脚踝、膝盖、臀部、肩膀、手腕等关键点的 y ≥ ground_y - ε 的数量 N_ground
  • 条件：H_ratio < 0.5 且 N_ground ≥ 3 → A 触发，贡献得分 0.35

B. 地面区域判定

  • 地面区域由预设的梯形/四边形 ROI 标定（摄像头安装角度固定时一次配置）
  • 条件：贴地关键点（ankles, knees, hips 中 y 最低的几个）的 (x, y) 必须落在地面 ROI 内
  • 满足 → B 触发，贡献得分 0.25

C. 由动到静 + 持续静止

  • 动：Track 质心在 T窗口 内的位移标准差 σ_motion
  • 由动到静：σ_motion(前 0.5s) > δ1 且 σ_motion(后 1.0s) < δ2
  • 持续静止：连续 ≥ M 帧（如 0.8s~1.5s）保持 σ_motion < δ2
  • 满足 → C 触发，贡献得分 0.40
规则总得分：
S_rule = 0.35·I(A) + 0.25·I(B) + 0.40·I(C)
权重可配置。T_trigger 建议初始值 0.60（A+B 同触发即 0.60，或 A+C 同触发 0.75）。

二、姿态分类器（二阶段验证 / 规则触发后运行）

输入：三通道特征融合

  1. 图像特征：当前帧人体 ROI（96×96 或 128×128），经 2~3 层轻量 Conv → 32-d 向量
  2. 关键点位姿特征：17 关键点坐标归一化到 [0,1] + 可见性得分 → 51-d 向量，经 1 层 FC → 32-d
  3. 运动特征：Track 的速度、加速度、高度压缩比 H_ratio、N_ground 等 → 8-d 向量

融合：concat(32 + 32 + 8 = 72-d) → 2 层 FC（72→32→1） → Sigmoid → S_cls ∈ [0,1]

三、融合决策器（最终判定）

决策公式（示例）：

S_final = α * S_rule + β * S_cls + γ * S_temporal
其中 α=0.35, β=0.45, γ=0.20（可配置）
  • S_temporal：过去 K 帧（如 5~10 帧）分类器得分的滑动平均，抑制单帧抖动
  • 报警条件：S_final ≥ T_alarm（建议初始 0.70）且连续满足 ≥ N_alarm 帧（建议 0.5s~1.0s）
  • 复位条件：连续 3s 内不满足报警条件，则清除跌倒状态
