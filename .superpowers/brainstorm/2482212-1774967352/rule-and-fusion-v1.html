二、17关键点说明

关键点分组

部位 关键点编号 用途
头部 0-4（鼻、眼、耳） 计算head_y
肩部 5-6 上身定位
上肢 7-10（肘、腕） 贴地检测
髋部 11-12 计算hip_y
膝部 13-14 贴地检测
踝部 15-16 计算lower_y

关键指标

h_ratio = (head_y - lower_y) / bbox_h
• 站立时 ≈ 0.85~0.95
• 跌倒时 < 0.6（大幅压缩）
hip_ratio = (head_y - hip_y) / bbox_h
• 判断站立vs坐姿
• 辅助姿态分类

三、5个判定规则

触发条件： S_rule ≥ 0.6（至少3个规则触发）

A. 高度压缩 + 多点贴地

最核心的规则，检测人体高度压缩和关键点贴地
权重: 1/5
条件1 h_ratio < 0.6 高度压缩明显
条件2 n_ground ≥ 2 至少2点贴地
条件3 visible_ratio ≥ 0.6 关键点可见性足够

B. 地面区域判定

检测最低关键点是否在地面区域
权重: 1/5
方案1（有预设ROI）
最低3个关键点是否在预设地面梯形/四边形内
方案2（无ROI）
最低关键点是否在bbox底部40%区域

C. 由动到静或持续静止

检测运动停止或躺卧状态持续静止
权重: 1/5
由动到静
avg_early > 75px/s
avg_late < 20px/s
持续静止
姿态 = lying
total_avg < 20px/s
窗口：1.5秒（前后半段对比）

D. 垂直快速下降

检测垂直方向快速下落速度
权重: 1/5
条件1 vy > 400 px/s 垂直下落速度大
条件2 h_ratio < 0.6 同时高度压缩
窗口：约5帧（0.2秒）

E. 加速度辅助

检测快速下落的加速度特征
权重: 1/5
条件1 accel > 150 px/s² 向下加速度大
条件2 h_ratio < 0.6 同时高度压缩
窗口：最近4帧计算速度差

规则得分计算

S_rule = (A + B + C + D + E) / 5
示例：
  • A+B触发 → 2/5 = 0.4（未达标）
  • A+B+C触发 → 3/5 = 0.6（达标 ✓）
惩罚：visible_ratio < 0.6 → 得分减半

八、分类器自适应权重

融合分类器

α = 0.35（规则权重）
β = 0.45（分类器权重）
alarm_thresh = 0.7

简单分类器

α = 0.15（规则权重↓）
β = 0.60（分类器权重↑）
alarm_thresh = 0.4（阈值↓）
简单分类器仅依赖图像，可靠性更高

流程总结

  1. 规则筛选：5规则等权平均 → S_rule ≥ 0.6触发分类器
  2. 姿态分类：h_ratio判定站/坐/蹲/躺，辅助序列检查
  3. 分类器验证：融合/简单分类器输出S_cls
  4. 融合得分：加权融合 + 时序平滑 → S_final ≥ 0.7
  5. 状态机决策：连续5帧 + 姿态序列 → FALLING告警
  6. 冷却防抖：5秒冷却 + 2秒恢复 → 防止重复告警
