端侧跌倒检测系统架构（方案A）

海思3516C（0.5T INT8 / 15M DDR / 30M Flash）

输入：1920×1080 @ 25fps → 抽帧（如 1/2 或 1/3）
↓
模块1：人体检测器
NanoDet / PP-YOLOE-s-MobileNet
输入 832×448 INT8
↓ 检测框 + 置信度
模块2：ByteTrack-lite
卡尔曼 + 匈牙利匹配
无 ReID 网络，输出 Track ID
↓ Track ROI
模块3：17关键点姿态估计
MobileNetV2 backbone
输入 96×96/128×128（基于检测框裁剪放大）
↓ 17 keypoints + 可见性得分
模块4：规则引擎（快速筛）
A 高度压缩 / B 地面区域 / C 由动到静
输出“疑似跌倒得分”
↓ 规则疑似触发 → 启动分类器二阶段验证
模块5：姿态分类器
融合 ROI 图像 + 17关键点坐标 + 运动速度特征
输出跌倒概率
↓
模块6：融合决策器
规则得分 + 分类器概率 + 时序一致性投票
最终输出 跌倒/非跌倒 事件
↓
输出：报警事件 + 目标框 + 关键点 + 跌倒置信度

核心设计要点

  • 抽帧策略：检测器每 2~3 帧跑一次（8~12 fps），跟踪器做预测补帧，关键点与分类器只在规则疑似触发时运行，降低平均算力消耗
  • 内存管理：检测模型（~2-3M）+ 关键点模型（~1.5M）+ 分类器（~0.5M），INT8 权重合计 < 6M，峰值 DDR 通过 imgsz 控制
  • 17关键点：采用 COCO 格式（nose, eyes, ears, shoulders, elbows, wrists, hips, knees, ankles），先用 COCO 预训练，再在场景数据上做 domain adaptation
