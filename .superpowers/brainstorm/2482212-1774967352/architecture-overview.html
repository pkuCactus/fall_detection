端侧跌倒检测系统架构（实际实现）

海思3516C（0.5T INT8 / 15M DDR / 30M Flash）

输入：1920×1080 @ 25fps → 抽帧（每3帧检测1次，skip_frames=2）
↓
模块1：人体检测器
YOLOv8n / YOLO-World
输入 832×448 INT8
权重 ≤ 3M
↓ 检测框 + 置信度
模块2：ByteTrack-lite
卡尔曼滤波 + IoU匹配
无 ReID 网络，输出 Track ID
耗时 ~2ms
↓ Track ROI（抽帧时预测补位）
模块3：17关键点姿态估计
YOLOv8n-pose backbone
COCO格式（nose, eyes, ears, shoulders, elbows, wrists, hips, knees, ankles）
权重 ≤ 2.5M，耗时 ~12ms
↓ 17 keypoints (x, y, conf) + 姿态分类
模块4：规则引擎（快速筛选）
A 高度压缩 + 多点贴地
B 地面区域判定
C 由动到静或持续静止
D 垂直快速下降
E 加速度辅助
输出 S_rule ∈ [0, 1]
↓ 规则疑似触发（S_rule ≥ trigger_thresh）→ 启动分类器
模块5：跌倒分类器（二选一）
方案A：融合分类器（ROI 96×96 + 17关键点 + 8维运动特征）
权重 ≤ 0.15M，耗时 ~3ms
方案B：简单分类器（仅96×96图像输入）
更轻量，适合边缘部署
↓ S_cls ∈ [0, 1]
模块6：融合决策器（状态机）
S_final = α·S_rule + β·S_cls + γ·S_temporal
状态机：NORMAL → SUSPECTED → FALLING → ALARM_SENT → RECOVERING
包含姿态序列检查、冷却期、恢复期机制
↓
输出：跌倒告警（Track ID + 置信度 + 姿态状态）

核心设计要点

  • 抽帧策略：检测器每3帧运行一次（skip_frames=2），跟踪器预测补帧，姿态估计和分类器只在规则触发后运行，降低平均算力消耗（总耗时 ~34ms）
  • 内存管理：YOLOv8n（~3M）+ YOLOv8n-pose（~2.5M）+ 分类器（~0.15M），INT8权重合计 < 6M，峰值DDR ~12M
  • 17关键点：采用COCO格式（nose, eyes, ears, shoulders, elbows, wrists, hips, knees, ankles），YOLOv8n-pose全帧推理后通过IoU匹配track bbox
  • 规则引擎：5个判定规则（A/B/C/D/E）等权平均，可见性不足时得分减半，包含姿态分类（standing/sitting/crouching/lying/unknown）
  • 分类器选择：支持融合分类器（3分支特征融合）和简单分类器（单分支图像分类），后者更轻量适合边缘部署
  • 状态机防抖：NORMAL→SUSPECTED→FALLING→ALARM_SENT→RECOVERING，连续N帧满足条件才告警，包含冷却期防止重复告警

实际性能基准（3516C模拟）

模块 耗时 内存
检测器 (YOLOv8n) ~15ms ~6M
跟踪器 (ByteTrack-lite) ~2ms ~0.5M
姿态估计 (YOLOv8n-pose) ~12ms ~5M
规则引擎 ~1ms 可忽略
分类器 ~3ms ~0.5M
融合决策 ~1ms 可忽略
总计 ~34ms (满足25fps) ~12M (满足15M DDR)

测试覆盖与训练流程

  • 测试覆盖：444个测试用例，单元测试覆盖率90%+，覆盖所有核心模块（detector/tracker/pose/rules/fusion/classifier/pipeline）
  • 训练流程：7个阶段训练（检测器→姿态→跟踪器调优→特征提取→融合分类器→简单分类器→端到端评估），支持DDP多GPU训练
  • 数据格式：YOLO格式（检测/姿态训练）、COCO格式（简单分类器）、私有labels.json（融合分类器）、event_gt.json（端到端评估）
  • 工具支持：VOC转YOLO转换器、YOLO自动标注、VLM自动标注、YOLO-World验证/导出工具
