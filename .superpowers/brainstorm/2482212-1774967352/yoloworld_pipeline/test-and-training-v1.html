测试流程 & 分阶段训练流程 — 方案2：YOLO-World Pipeline

开放词汇检测 + 运动/宽高比辅助 + 状态机融合 | 简化训练流程

一、整体测试流程（Test Pipeline）

测试统计

  • 总测试用例：444个（与方案1共用大部分单元测试，pipeline 层独立测试）
  • 单元测试覆盖率：90%+（detector/tracker/fusion 共用，pipeline 独立）
  • 运行方式：PYTHONPATH=src pytest tests/unit/test_pipeline.py -v（需区分 pipeline 类型）

阶段1：模块级单元测试（复用 + 新增）

  • 检测器：test_detector.py - YOLO-World 模型加载、开放词汇类别输出、imgsz 适配
  • 跟踪器：test_tracker.py - ByteTrack-lite 共用
  • 融合决策：test_fusion.py - FusionDecision 共用
  • YOLO-World Pipeline：test_yoloworld_pipeline.py - 抽帧逻辑、det_info 缓存、跌倒得分计算、运动/宽高比 bonus
  • 几何工具：test_utils_geometry.py - iou 计算（用于匹配检测框到轨迹）

阶段2：端到端 Inference Pipeline 测试

  • 视频级测试：scripts/eval/evaluate_pipeline.py - 给定1080p视频，输出跌倒事件检测结果
  • 评估指标：事件级TP/FP/FN、漏报率、误报率，对比 event_gt.json 标注
  • 实时性能：scripts/eval/benchmark_speed.py - FPS、延迟P50/P99
  • 阈值搜索：网格搜索 alarm_thresh / cls_bypass_thresh 最优组合

阶段3：端侧模拟部署验证

  • 模型转换：PyTorch → ONNX，scripts/tools/export_yolo_world.py 支持非正方形分辨率（832×448）
  • 量化验证：INT8量化前后精度对比（检测mAP、姿态分类 accuracy）
  • 资源评估：峰值DDR（~6.5M）、模型加载flash大小、每帧前向耗时 benchmark

二、简化训练流程（Training Pipeline）

核心原则：YOLO-World 开放词汇检测器支持零样本/少样本姿态类别检测，无需独立姿态估计器和分类器训练。

DDP支持：YOLO-World 训练脚本支持多GPU分布式训练，使用 --ngpus N 参数。

配置文件：configs/training/yoloworld.yaml 提供训练参数配置。

阶段1：YOLO-World 检测器训练/微调

数据集：YOLO格式（COCO person + 私有跌倒多姿态标注）

脚本：scripts/train/train_yoloworld.py

Shell：bash scripts/shell/run_train_yoloworld.sh --ngpus 2

配置：configs/training/yoloworld.yaml

  1. YOLO-World 预训练初始化（开放词汇能力）
  2. 微调多姿态类别：standing/sitting/crouching/lying/fallen
  3. 输入 832×448，支持非正方形分辨率
  4. 支持DDP多GPU训练

输出：outputs/yoloworld/best.pt

阶段2：跟踪器参数调优

方式：纯调参，不训练神经网络

脚本：scripts/eval/tune_tracker.py

Shell：bash scripts/shell/run_tune_tracker.sh --video-dir data/videos

  1. 在 YOLO-World 检测输出的视频序列上运行 ByteTrack-lite
  2. 网格搜索：track_thresh / match_thresh / max_age / min_hits
  3. 评估 MOTA / IDF1，确认类别切换（class switch）可接受

输出：outputs/tracker/tune_result.json

阶段3：跌倒得分配置调优

方式：配置调优，无需神经网络训练

配置：configs/pipeline/yolo_world.yaml

  1. 配置 fall_scores：映射类别到基础跌倒分（0.0~1.0）
  2. 配置 posture_map：映射类别到姿态标签（用于状态机序列检查）
  3. 调整 motion_window_frames、运动阈值
  4. 调整 aspect 阈值和 bonus 系数

输出：优化后的 yolo_world.yaml

阶段4：端到端评估与阈值搜索

脚本：scripts/eval/evaluate_pipeline.py --pipeline yoloworld

Shell：bash scripts/shell/run_evaluate_yoloworld_pipeline.sh

  1. 冻结 YOLO-World 权重，在验证集上跑完整 pipeline
  2. 网格搜索阈值：alarm_thresh [0.5, 0.6, 0.7]、cls_bypass_thresh [0.9, 0.95, 1.0]
  3. 评估事件级 TP/FP/FN，计算最优阈值组合
  4. 输出最终部署模型 + 配置文件

输出：outputs/eval/yoloworld_eval_result.json

三、方案2 vs 方案1 训练流程对比

方案1：Keypoint Pipeline（7阶段）

  1. 检测器训练（YOLOv8n）
  2. 姿态估计器微调（YOLOv8n-pose）
  3. 跟踪器调优
  4. 特征提取（ROI+关键点+运动）
  5. 融合分类器训练（3分支）
  6. 简单分类器训练（单分支）
  7. 端到端评估

方案2：YOLO-World Pipeline（4阶段）

  1. YOLO-World 检测器训练/微调
  2. 跟踪器调优
  3. 跌倒得分配置调优
  4. 端到端评估

省去姿态估计器、特征提取、融合分类器、简单分类器四个训练阶段，训练周期大幅缩短。

训练可复现性保障

  • 配置文件：configs/training/yoloworld.yaml 记录所有训练参数
  • 随机种子：支持 seed 参数（默认666）
  • Checkpoint保存：保存 best.pt 和 last.pt
  • 训练日志：outputs/yoloworld/train.log 记录 loss/mAP 曲线
