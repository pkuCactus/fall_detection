运动辅助：垂直快速下降（motion_bonus）

检测垂直方向快速下落，跌倒过程通常伴随明显的 vy 增大
vy = centers[-1].y - centers[-2].y
if vy > 50:
motion_bonus = min(0.5, vy / 400.0)
• vy 单位：像素/帧（原始位移，未乘 fps）
• 上限 0.5，避免极端速度导致得分溢出

宽高比辅助：倒下的人更宽（aspect_bonus）

跌倒后人体通常横向展开，bbox 宽高比明显大于正常直立状态
aspect = w / max(h, 1.0)
if aspect > 1.2:
aspect_bonus = min(0.2, (aspect - 1.2) × 0.3)
• 正常直立 aspect ≈ 0.3~0.6
• 躺下 aspect 通常 > 1.2，上限 0.2

高置信度加速通道（cls_bypass_thresh）
当 S_fall ≥ cls_bypass_thresh（默认1.0=禁用）时：
• 绕过姿态序列检查，直接进入 FALLING
• alarm_min_frames 减半（最小为2）
冷却期机制（cooldown_seconds，默认5秒）
进入 ALARM_SENT 后，cooldown_frames 内不重复告警。

六、姿态序列检查

检查最近 sequence_check_frames 帧（默认8帧）姿态历史

检查要求（四项必须全部满足）：
  • 最近窗口内包含 upright 姿态（standing 或 sitting）
  • 最近窗口内包含 fall 姿态（crouching 或 lying）
  • 当前帧姿态为 fall 姿态
  • fall 姿态数量 严格大于 upright 姿态数量（fall majority）
防误判：一直躺着（无转换）→ 不告警。
Fallback 机制：若长期在 SUSPECTED 且未恢复（above_count ≥ max(sequence_check_frames, alarm_min_frames×2)），且当前姿态为 crouching/lying/unknown，则允许直接进入 FALLING。

流程总结

  1. 开放词汇检测：YOLO-World 直接输出多姿态类别（standing/sitting/crouching/lying/fallen）
  2. 跟踪关联：ByteTrack-lite 跟踪 + IoU 匹配检测框类别到轨迹
  3. 跌倒得分：fall_scores 映射基础分 × det_conf + motion_bonus × 0.1 + aspect_bonus
  4. 融合得分：S_fall 同时作为 rule_score 和 cls_score，+ S_temporal 时序平滑 → S_final
  5. 状态机决策：enter_suspected_min_frames → SUSPECTED → alarm_min_frames + 姿态序列检查 → FALLING告警
  6. 冷却防抖：cooldown_seconds 冷却 + recovery_seconds 恢复 → 防止重复告警
