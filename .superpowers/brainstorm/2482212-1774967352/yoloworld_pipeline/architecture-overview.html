端侧跌倒检测系统架构 — 方案2：YOLO-World Pipeline（实际实现）

海思3516C（0.5T INT8 / 15M DDR / 30M Flash）| YOLO-World 开放词汇 + 运动/宽高比辅助 + 状态机融合

输入：1920×1080 @ 25fps → 抽帧（每3帧检测1次，skip_frames=2）
↓
模块1：YOLO-World 开放词汇检测器
多姿态类别同时检测（standing/sitting/crouching/lying/fallen...）
输入 832×448 INT8
权重 ≤ 3M，模型类型 yolo_world
↓ 检测框 + 类别名 + 置信度
模块2：ByteTrack-lite
卡尔曼滤波 + IoU匹配（高置信度优先，低置信度二次匹配）
无 ReID 网络，Track 状态：tentative → confirmed
IoU 关联检测类别到轨迹（_match_detections_to_tracks）
耗时 ~2ms
↓ Track + 检测类别信息（抽帧时复用 last_det_info）
模块3：跌倒得分计算（无规则引擎，无分类器）
base_score = fall_scores[class_name]（配置映射）
motion_bonus = 垂直快速下降 vy（vy > 50 → min(0.5, vy/400)）
aspect_bonus = 宽高比（aspect > 1.2 → min(0.2, (aspect-1.2)×0.3)）
score = base_score × det_conf + motion_bonus × 0.1 + aspect_bonus
↓ S_fall ∈ [0, 1]，同时作为 rule_score 和 cls_score
模块4：融合决策器 FusionDecision（状态机）
S_final = α·S_fall + β·S_fall + γ·S_temporal
状态机：NORMAL → SUSPECTED → FALLING → ALARM_SENT → RECOVERING
cls_bypass_thresh 高置信度绕过姿态序列检查并减半 alarm_min_frames
姿态序列检查要求：upright → fall 转换 + fall majority
↓
输出：跌倒告警（Track ID + 置信度 + 姿态状态 + debug信息）

核心设计要点

  • 无姿态估计器：YOLO-World 直接输出多姿态类别，无需 YOLOv8n-pose 和 17 关键点提取，降低算力和内存。
  • 无规则引擎：不依赖 h_ratio、n_ground、地面ROI 等几何规则，直接利用检测器语义输出。
  • 无分类器：不运行 FallClassifier 或 SimpleFallClassifier，进一步减少模型加载和推理开销。
  • 跌倒得分计算：由配置 fall_scores 映射类别到基础分，乘以检测置信度，叠加运动 bonus 和宽高比 bonus。
  • 运动辅助：基于 track 中心点历史计算垂直速度 vy，快速下降时给予 motion_bonus（vy/400，上限0.5）。
  • 宽高比辅助：倒下的人 bbox 通常 wider than tall（aspect > 1.2），给予 aspect_bonus（上限0.2）。
  • 状态机复用：与方案1共用 FusionDecision 状态机，同样具备姿态序列检查、cls_bypass、冷却期、恢复期机制。
  • 抽帧策略：检测器每3帧运行一次，skip帧时 tracker 预测补位，复用上一帧 det_info 维持类别标签。
