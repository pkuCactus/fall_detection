二、17关键点说明

关键点分组

部位关键点编号用途
头部0-4（鼻、眼、耳）计算head_y
肩部5-6上身定位、torso_angle
上肢7-10（肘、腕）贴地检测、水平展开
髋部11-12计算hip_y、hip_ratio
膝部13-14贴地检测
踝部15-16计算lower_y

关键指标

h_ratio = |head_y - lower_y| / bbox_h
• 站立时 ≈ 0.85~0.95
• 跌倒时 < 0.6

hip_ratio = |head_y - hip_y| / bbox_h
• 判断站立vs坐姿（坐姿 hip_ratio 小）
• 辅助姿态分类、防坐姿误判

kpt_aspect = kpt_span_w / kpt_span_h
• 光轴方向倒下检测：> 1.5 → lying

torso_angle = atan2(dx, dy) 度
• 躯干接近水平：> 55° → lying

三、6个判定规则

触发逻辑： S_rule = (A+B+C+D+E+F) / 6 visible_ratio < 0.6 且非 F 触发 → 得分减半

A. 高度压缩 + 多点贴地（含光轴方向倒下）

最核心规则，检测人体高度压缩、关键点贴地、光轴方向倒下
权重: 1/6
条件1h_ratio < 0.6高度压缩明显
条件2n_ground ≥ 2至少2点贴地
条件3visible_ratio ≥ 0.6关键点可见性足够
光轴辅助kpt_aspect > 1.5 或 torso_angle > 55°向/远离相机倒下
防误判约束：
• 坐姿且 hip_ratio < 0.45 且 torso_angle ≤ 45° → 强制不触发A
• 脚踝不可见但 hip_ratio > 0.7 且 kpt_aspect ≤ 1.2 且 torso_angle ≤ 45° → 强制不触发A

B. 地面区域判定

检测最低关键点是否在地面区域
权重: 1/6
方案1（有预设ROI）
最低3个可见关键点是否全部在预设地面多边形内
方案2（无ROI）
最低关键点是否在 bbox 底部 40% 区域（ground_y_thresh），至少1个点满足

C. 由动到静或持续静止

检测运动停止或躺卧状态持续静止
权重: 1/6
由动到静
avg_early > 75 px/s
avg_late < 20 px/s
持续静止
姿态 = lying
total_avg < 20 px/s
窗口：1.5秒（前后半段对比），位移单位 px/s（已乘fps归一化）

D. 垂直快速下降

检测垂直方向快速下落速度
权重: 1/6
条件1vy > 400 px/s垂直下落速度大
条件2h_ratio < 0.6 或 posture == lying同时高度压缩或已躺卧
窗口：约0.2秒（max(3, 0.2*fps) 帧），vy 单位 px/s

E. 加速度辅助

检测快速下落的加速度特征
权重: 1/6
条件1accel > 150 px/s²向下加速度大
条件2h_ratio < 0.6 或 posture == lying同时高度压缩或已躺卧
窗口：最近4帧计算速度差，accel = (vys[-1] - vys[0]) / (len(vys)/fps)

F. 关键点完全不可见

检测异常遮挡或极端姿态导致关键点全失
权重: 1/6
条件visible_ratio < 0.1关键点几乎完全不可见
规则F触发时不执行可见性惩罚减半（因为F本身就是检测不可见场景）

光轴方向倒下检测

• kpt_aspect > 1.5：关键点水平展开远超垂直 → 判 lying
• torso_angle > 55°：肩-髋连线接近水平 → 判 lying
• bbox_aspect > 1.3 且 kpt_aspect > 1.0 且 h_ratio < 0.65：检测框变宽 → 判 lying

分类器得分辅助姿态修正（cls_posture_t1 / cls_posture_t2）

• cls_score > t1：强制修正为 lying（高置信度跌倒）
• cls_score < t2：强制修正为 standing（高置信度非跌倒）
• t2 ≤ cls_score ≤ t1：若原姿态为 standing，降级为 sitting
仅当 t1/t2 在 [0, 1] 范围内时生效；默认 t1=1.0, t2=-1.0（即默认禁用）

七、姿态序列检查

检查最近 sequence_check_frames=8 帧姿态历史

检查要求（四项必须全部满足）：
  • 最近窗口内包含 upright 姿态（standing 或 sitting）
  • 最近窗口内包含 fall 姿态（crouching 或 lying）
  • 当前帧姿态为 fall 姿态
  • fall 姿态数量 严格大于 upright 姿态数量（fall majority）
防误判：一直躺着（无转换）→ 不告警；一直站着（无转换）→ 不告警
Fallback 机制：若长期在 SUSPECTED 且未恢复（above_count ≥ max(8, 5×2)=10），且当前姿态为 crouching/lying/unknown，则允许直接进入 FALLING，避免高置信度场景被序列检查卡住。

八、分类器自适应权重

α = 0.35（规则权重）
β = 0.45（分类器权重）

简单分类器（自动调整）

α = 0.105（规则权重自动×0.3）

alarm_thresh = 0.56（自动×0.8）

流程总结

  1. 规则筛选：6规则等权平均 → S_rule，含光轴方向倒下检测、坐姿防误判
  2. 姿态分类：h_ratio + kpt_aspect + torso_angle + cls_score 联合判定站/坐/蹲/躺
  3. 分类器验证：融合/简单分类器输出 S_cls，简单分类器自动调整融合权重
  4. 融合得分：加权融合 + 时序平滑 → S_final ≥ 0.70
  5. 状态机决策：3帧 → SUSPECTED → 5帧 + 姿态序列检查（fall majority）→ FALLING告警，cls_bypass可加速
  6. 冷却防抖：cooldown_seconds=5.0s 冷却 + recovery_seconds=2.0s 恢复 → 防止重复告警
