端侧跌倒检测系统架构 — 方案1：Keypoint Pipeline（实际实现）

海思3516C（0.5T INT8 / 15M DDR / 30M Flash）| YOLOv8n + 17关键点 + 规则引擎 + 分类器融合

输入：1920×1080 @ 25fps → 抽帧（每3帧检测1次，skip_frames=2）
↓
模块1：人体检测器
YOLOv8n
输入 832×448 INT8
权重 ≤ 3M
↓ 检测框 + 置信度
模块2：ByteTrack-lite
卡尔曼滤波 + IoU匹配（高置信度优先，低置信度二次匹配）
无 ReID 网络，Track 状态：tentative → confirmed
耗时 ~2ms
↓ Track ROI（抽帧时预测补位，可选 SimpleKeypointTracker 光流预测）
模块3：17关键点姿态估计
YOLOv8n-pose 全帧推理 → IoU匹配 track bbox
COCO格式（nose, eyes, ears, shoulders, elbows, wrists, hips, knees, ankles）
权重 ≤ 2.5M，耗时 ~12ms
↓ 17 keypoints (x, y, conf) + 姿态分类（standing/sitting/crouching/lying/unknown）
模块4：规则引擎（6规则快速筛选）
A 高度压缩 + 多点贴地（含光轴方向倒下检测）
B 地面区域判定（ROI 或 bbox底部40%）
C 由动到静或持续静止
D 垂直快速下降
E 加速度辅助
F 关键点完全不可见（异常触发）
输出 S_rule ∈ [0, 1]，接收 cls_score 辅助姿态分类
↓ 规则疑似触发（S_rule ≥ trigger_thresh）→ 启动分类器
模块5：跌倒分类器（二选一）
方案A：融合分类器（ROI 96×96 + 17关键点 + 8维运动特征）
权重 ≤ 0.15M，耗时 ~3ms
方案B：简单分类器（仅96×96图像输入，2-class softmax）
更轻量，适合边缘部署
↓ S_cls ∈ [0, 1]
模块6：融合决策器 FusionDecision（状态机）
S_final = α·S_rule + β·S_cls + γ·S_temporal
状态机：NORMAL → SUSPECTED → FALLING → ALARM_SENT → RECOVERING
cls_bypass_thresh 高置信度绕过姿态序列检查并减半 alarm_min_frames
姿态序列检查要求：站/坐 → 跌 转换 + fall 姿态 majority
enter_suspected_min_frames 防止瞬态抖动进入 SUSPECTED
↓
输出：跌倒告警（Track ID + 置信度 + 姿态状态 + 规则flags）

核心设计要点

  • 抽帧策略：检测器每3帧运行一次（skip_frames=2），跟踪器预测补帧；姿态估计和分类器每帧都推理（skip帧复用/预测关键点）。总耗时 ~34ms。
  • 可选关键点跟踪器：SimpleKeypointTracker 支持 alpha 平滑、速度衰减、光流预测（默认关闭，避免闪烁）。
  • 内存管理：YOLOv8n（~3M）+ YOLOv8n-pose（~2.5M）+ 分类器（~0.15M），INT8权重合计 < 6M，峰值DDR ~12M。
  • 17关键点：COCO格式，全帧推理后通过IoU(阈值0.1)匹配track bbox，未匹配bbox得零关键点。
  • 规则引擎：6规则等权平均（A/B/C/D/E/F），cls_score辅助姿态分类（cls_posture_t1/t2），光轴方向倒下检测（kpt_aspect/torso_angle），坐姿防误判（hip_ratio约束）。
  • 分类器选择：融合分类器（3分支特征融合）或简单分类器（单分支图像分类）；简单分类器自动降低融合alpha、提高beta、降低alarm_thresh。
  • 状态机防抖：NORMAL→SUSPECTED（需连续 enter_suspected_min_frames=3 帧超阈值）→FALLING（alarm_min_frames=5 + 姿态序列检查，cls_bypass可绕过）→ALARM_SENT→RECOVERING，冷却期 cooldown_seconds=5.0s 防止重复告警。

实际性能基准（3516C模拟）

模块耗时内存

测试覆盖与训练流程

  • 测试覆盖：444个测试用例，单元测试覆盖率90%+，覆盖所有核心模块（detector/tracker/pose/rules/fusion/classifier/pipeline）
  • 训练流程：7个阶段训练（检测器→姿态→跟踪器调优→特征提取→融合分类器→简单分类器→端到端评估），支持DDP多GPU训练
  • 数据格式：YOLO格式（检测/姿态训练）、COCO格式（简单分类器）、私有labels.json（融合分类器）、event_gt.json（端到端评估）
  • 工具支持：VOC转YOLO转换器、YOLO自动标注、VLM自动标注、YOLO-World验证/导出工具
