测试流程 & 分阶段训练流程 — 方案1：Keypoint Pipeline

444个测试用例 + 7阶段训练流程 + DDP多GPU支持

一、整体测试流程（Test Pipeline）

测试统计

  • 总测试用例：444个（unit tests + integration tests）
  • 单元测试覆盖率：90%+（核心模块detector/tracker/pose/rules/fusion/classifier/pipeline）
  • 运行方式：bash scripts/shell/run_tests.sh 或 PYTHONPATH=src pytest tests/ -v
  • 覆盖率检查：PYTHONPATH=src pytest tests/ --cov=src/fall_detection --cov-fail-under=90

阶段1：模块级单元测试（16个单元测试文件）

  • 检测器：test_detector.py - 检测输出格式、置信度阈值、模型加载
  • 跟踪器：test_tracker.py - ByteTrack-lite跟踪、ID分配、状态转换、卡尔曼滤波
  • 关键点：test_pose_estimator.py - 姿态估计输出、关键点匹配、IoU关联
  • 规则引擎：test_rules.py - 六个判定规则（A/B/C/D/E/F）、姿态分类、cls_score辅助、得分计算
  • 融合决策：test_fusion.py - 状态机流转、姿态序列检查（fall majority）、cls_bypass、冷却期机制
  • 分类器：test_classifier.py、test_simple_classifier.py - 融合/简单分类器推理、权重加载
  • Pipeline：test_pipeline.py - 抽帧逻辑、缓存机制、关键点跟踪器、端到端处理
  • 数据处理：test_augmentation.py、test_datasets.py - 数据增强、数据集加载
  • 工具：test_convert_voc_to_yolo.py、test_export.py - VOC转换、模型导出
  • 辅助工具：test_utils_*.py、test_scheduler.py - 可视化、几何计算、调度器

阶段2：集成测试（6个集成测试文件）

  • 训练脚本测试：test_training_detector.py、test_training_pose.py、test_training_classifier.py、test_training_simple_classifier.py - DDP训练、参数解析、模型保存
  • 工具脚本测试：test_training_scripts.py、test_extract_features.py - 特征提取、脚本help信息

阶段3：端到端 Inference Pipeline 测试

  • 视频级测试：scripts/eval/evaluate_pipeline.py - 给定1080p视频，输出跌倒事件检测结果
  • 评估指标：事件级TP/FP/FN、漏报率、误报率，对比event_gt.json标注
  • 实时性能：scripts/eval/benchmark_speed.py - FPS、延迟P50/P99、不同抽帧率下的端到端延迟
  • 阈值搜索：scripts/shell/run_evaluate_pipeline.sh - 网格搜索trigger_thresh/alarm_thresh最优组合

阶段4：端侧模拟部署验证

  • 模型转换：PyTorch → ONNX，src/fall_detection/utils/export.py提供导出接口
  • YOLO-World导出：scripts/tools/export_yolo_world.py支持非正方形分辨率（832×448）
  • 量化验证：INT8量化前后精度对比（检测mAP、关键点PCK、分类器accuracy）
  • 资源评估：峰值DDR、模型加载flash大小、每帧前向耗时benchmark

二、7阶段训练流程（Training Pipeline）

核心原则：检测器和姿态估计器独立预训练；跟踪器基于检测结果调参；分类器串行训练；最后端到端串联评估。

DDP支持：所有训练脚本支持多GPU分布式训练，使用--ngpus N参数。

配置文件：configs/training/*.yaml提供训练参数配置（detector.yaml、pose.yaml、classifier.yaml、simple_classifier.yaml等）。

阶段1：人体检测器训练

数据集：YOLO格式（COCO person + 私有跌倒人体框）

脚本：scripts/train/train_detector.py

Shell：bash scripts/shell/run_train_detector.sh --ngpus 2

配置：configs/training/detector.yaml

  1. YOLOv8n预训练初始化
  2. 私有数据fine-tune，输入832×448
  3. 支持DDP多GPU训练

输出：outputs/detector/best.pt

阶段2：姿态估计器微调

数据集：YOLO格式（COCO keypoints 17点标注）

脚本：scripts/train/train_pose.py

Shell：bash scripts/shell/run_train_pose.sh --ngpus 2

配置：configs/training/pose.yaml

  1. YOLOv8n-pose预训练初始化
  2. 输入128×96或256×192（权衡精度与DDR）
  3. 可补充跌倒场景关键点标注做domain adaptation

输出：outputs/pose/best.pt

阶段3：跟踪器参数调优

方式：纯调参，不训练神经网络

脚本：scripts/eval/tune_tracker.py

Shell：bash scripts/shell/run_tune_tracker.sh --video-dir data/videos

  1. 在阶段1检测器输出的视频序列上运行ByteTrack-lite
  2. 网格搜索：track_thresh/match_thresh/max_age/min_hits
  3. 评估MOTA/IDF1，确认ID Switch可接受

输出：outputs/tracker/tune_result.json

阶段4：特征提取

目的：为融合分类器准备训练数据

脚本：scripts/tools/extract_features.py

Shell：bash scripts/shell/run_extract_features.sh --video-dir data/videos --label-file data/labels.json

  1. 用阶段1检测器 + 阶段2姿态估计器提取特征
  2. ROI图像（96×96）+ 关键点坐标（17×3）+ 运动特征（8-d：vx,vy,ax,ay,w,h,h_ratio,n_ground）
  3. 缓存为.npz文件，避免重复前向推理

输出：outputs/cache/*.npz

阶段5：融合分类器训练

数据：outputs/cache/*.npz + data/labels.json

脚本：scripts/train/train_classifier.py

Shell：bash scripts/shell/run_train_classifier.sh --ngpus 2 --batch-size 16

配置：configs/training/classifier.yaml

  1. 3分支网络（图像ROI 96×96 + 17关键点 + 8维运动）
  2. 训练100~200 epochs，Dropout防过拟合
  3. 难例挖掘，补充非跌倒难例数据

输出：outputs/classifier/best.pt

阶段6：简单分类器训练

数据：COCO/VOC格式（跌倒/非跌倒图像bbox标注）

脚本：scripts/train/train_simple_classifier.py

Shell：bash scripts/shell/run_train_simple_classifier.sh --config configs/training/simple_classifier.yaml --ngpus 2

配置：configs/training/simple_classifier.yaml或simple_classifier_voc.yaml

  1. 单分支轻量CNN（仅96×96图像输入）
  2. 2-class分类（normal/fall），softmax输出
  3. 数据增强、早停、学习率warmup

输出：outputs/simple_classifier/best.pt

阶段7：端到端评估与阈值搜索

脚本：scripts/eval/evaluate_pipeline.py

Shell：bash scripts/shell/run_evaluate_pipeline.sh --video-dir data/videos --gt-file data/event_gt.json

  1. 冻结所有模块权重，在验证集上跑完整pipeline
  2. 网格搜索阈值：trigger_thresh [0.5, 0.6, 0.7]、alarm_thresh [0.6, 0.7, 0.8]
  3. 评估事件级TP/FP/FN，计算最优阈值组合
  4. 输出最终部署模型 + 配置文件

输出：outputs/eval/eval_result.json

一键运行完整训练流程

NGPUS=2 bash scripts/shell/run_all_training.sh

自动执行阶段1→7，输出所有训练产物。

三、数据格式与工具支持

数据格式

  • 检测/姿态训练：YOLO格式（images/ + labels/*.txt + data.yaml）
  • 简单分类器训练：COCO/VOC格式（images/ + annotations/*.json/xml）
  • 融合分类器标签：data/labels.json（video_name → label + frames）
  • 端到端评估GT：data/event_gt.json（video_name → [start, end]时间段列表）

工具支持

  • VOC转YOLO：scripts/tools/convert_voc_to_yolo.py - 支持YOLO-World风格映射
  • YOLO自动标注：tools/annotate/yolo_annotate.py - YOLOv8人体检测生成VOC标注
  • VLM自动标注：tools/annotate/vlm_annotate.py - Claude/阿里百炼API分析图像生成标注
  • YOLO-World工具：scripts/shell/run_validate_yolo_world.sh、run_export_yolo_world.sh - 验证/导出工具
  • Requirements生成：scripts/tools/generate_requirements.py - 指定CUDA版本生成requirements.txt

训练可复现性保障

  • 配置文件：configs/training/*.yaml记录所有训练参数
  • 随机种子：所有训练脚本支持seed参数（默认666）
  • Checkpoint保存：每个阶段保存best.pt和last.pt
  • 训练日志：outputs/*/train.log记录loss/accuracy曲线
