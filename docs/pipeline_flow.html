跌倒检测 Pipeline 全流程详解

1. Pipeline 整体流程

核心设计理念
采用 skip-frame 机制：检测帧每 N+1 帧运行一次（默认 N=2，即每3帧检测），跳帧时仅预测轨迹和推理分类器。 目标：满足边缘设备 ~34ms 总延迟预算（HiSilicon 3516C，15M DDR）。

2. 目标跟踪详解（ByteTrackLite）

ByteTrackLite 设计
裁剪版 ByteTrack，仅用 IoU 匹配，无外观特征（ReID）。适合边缘设备，减少计算开销。
核心思想：两阶段匹配，高分检测框优先匹配，低分检测框补充匹配，提高遮挡场景鲁棒性。

2.1 跟踪流程步骤

1
预测已有轨迹（Kalman Filter）
对所有 tracks 执行 predict()，更新位置和速度估计，age/time_since_update 递增
2
分离高/低分检测框
dets_high: conf ≥ track_thresh (0.5)，dets_low: conf < track_thresh
3
第一次匹配：active tracks + 高分检测框
active_tracks = confirmed + tentative，使用 IoU cost matrix + Hungarian 算法匹配
4
第二次匹配：未匹配 tracks + 低分检测框
为第一次未匹配的轨迹尝试与低分检测框关联，减少遮挡丢失
5
标记未匹配轨迹为丢失
调用 mark_missed()，tentative 直接删除，confirmed 若 time_since_update > max_age (30帧) 则删除
6
新建轨迹
未匹配的高分检测框 → 新建 tentative track，分配新 track_id

2.2 两阶段匹配详解

第一阶段：高分匹配
# 匹配对象
tracks: confirmed + tentative
dets: conf ≥ track_thresh (0.5)

# 匹配算法
cost = 1.0 - IoU(track.bbox, det.bbox)
matches = Hungarian(cost_matrix)

# 匹配条件
cost ≤ match_thresh (0.2)
→ IoU ≥ 0.8
目的：高置信度检测框优先关联，确保稳定跟踪
第二阶段：低分补充
# 匹配对象
tracks: 第一阶段未匹配的
dets: conf < track_thresh

# 同样使用 IoU 匹配
cost = 1.0 - IoU(...)
matches = Hungarian(cost_matrix)
目的：低分检测框挽救未匹配轨迹，处理遮挡/模糊场景

上一帧状态

→

增加不确定性

→

检测框

measurement

→

修正估计

# Predict: 状态转移
mean = F @ mean  # F: 状态转移矩阵
cov = F @ cov @ F.T + Q  # Q: 过程噪声

# Update: 观测修正
K = cov @ H.T @ inv(H @ cov @ H.T + R)  # Kalman增益
mean = mean + K @ (z - H @ mean)  # z: 检测框测量
cov = cov - K @ H @ cov

2.5 IoU Cost Matrix

cost_matrix[i,j] = 1.0 - IoU(track_i.bbox, det_j.bbox)
IoU ≥ 0.8 时 cost ≤ 0.2，满足 match_thresh 条件
def iou_cost(tracks, detections):
    cost_matrix = np.zeros((len(tracks), len(detections)))
    for i, track in enumerate(tracks):
        for j, det in enumerate(detections):
            cost_matrix[i, j] = 1.0 - iou(track.to_tlbr(), det.bbox)
    return cost_matrix

3. 规则引擎详解（RuleEngine）

规则得分计算
S_rule = (A + B + C + D + E + F) / 6，每条规则等权贡献，触发越多分数越高。
若 visible_ratio < visible_ratio_min 且 F 未触发，则 score *= 0.5（可见性惩罚）。
A 高度压缩 + 多点贴地
检测人体高度压缩（躺卧特征）+ 关键点贴地数量
flags["A"] = is_compressed AND (n_ground ≥ 2) AND (visible_ratio ≥ 0.5) is_compressed = (h_ratio < 0.6) OR (kpt_aspect > 1.5) OR (torso_angle > 55°)
h_ratio|head_y - ankle_y| / bbox_h，站立≈0.85，躺卧≈0.3
kpt_aspect关键点水平宽度/垂直高度，>1.5表示水平展开
torso_angle肩膀-臀部连线与垂直方向夹角，>55°表示躯干倾斜
n_groundbbox底部40%区域内的关键点数量
排除条件：若 posture=="sitting" 且 hip_ratio<0.45 且 torso_angle≤45°，强制 flags["A"]=False
B 地面 ROI 判定
检测最低关键点是否位于地面区域
if ground_roi is None: flags["B"] = n_near_ground ≥ 1 else: flags["B"] = all(point_in_polygon(p, ground_roi))
ground_roi可选：预定义地面多边形区域
lowest3置信度最高的3个最低关键点
ground_y_threshbbox底部40%边界
C 由动到静 / 持续静止
检测运动轨迹的变化：快速运动后突然静止
displacements[i] = sqrt(dx² + dy²) * fps # px/s flags["C"] = (avg_early > 50 px/s) AND (avg_late < 20 px/s) # 或躺卧持续静止 if posture == "lying": flags["C"] = total_avg < 20 px/s
motion_thresh运动期阈值 50 px/s
static_thresh静止期阈值 20 px/s
motion_window历史窗口 1.5秒
D 垂直方向快速下降
检测人体中心点垂直方向的快速下移速度
vy_px_s = (recent_y[-1] - recent_y[0]) / (window-1) * fps flags["D"] = (vy > 200 px/s) AND ((h_ratio < 0.6) OR (posture == "lying"))
fall_vy_thresh垂直速度阈值 200 px/s
window计算窗口 ≈0.2秒
注意：需同时满足姿态压缩，避免正常下坡误触发
E 向下加速度辅助
检测垂直方向的加速度特征
vys[i] = (y[i] - y[i-1]) * fps accel = (vys[-1] - vys[0]) / (len(vys) / fps) flags["E"] = (accel > 100 px/s²) AND (posture in {"lying", "compressed"})
accel_thresh加速度阈值 100 px/s²
F 关键点不可见检测
检测关键点几乎完全不可见（极端姿态或遮挡）
visible_ratio = sum(kpts[:,2] > 0.1) / 17 flags["F"] = visible_ratio < 0.1 # <17%可见
场景：跌倒后被遮挡、极端姿势导致姿态估计失败

4.1 姿态判定优先级

# 优先检测光轴方向跌倒（水平展开）
if kpt_aspect > 1.5 or torso_angle > 55: posture = "lying"

# 检测框明显变宽 + 压缩
elif bbox_aspect > 1.3 and kpt_aspect > 1.0 and h_ratio < 0.65: posture = "lying"

# 按高度压缩程度分层
elif h_ratio > 0.75: posture = "standing"
elif h_ratio > 0.50: posture = "sitting" (if hip_ratio < 0.45)
elif h_ratio > 0.35: posture = "crouching"
else: posture = "lying"

4.2 分类器得分辅助修正

条件修正动作

cls_score < cls_posture_t2 (0.2)强制 posture = "standing"
cls_score 在 [t2, t1] 区间若原为"standing"，降级为"sitting"

5. 融合决策详解（FusionDecision）

5.1 融合得分公式

S_final = α × S_rule + β × S_cls + γ × S_temporal
默认权重：α=0.35（规则）, β=0.45（分类器）, γ=0.20（时序平滑）
S_temporal = 最近 fps 帧分类器得分的滑动平均

5.4 姿态序列检查

检查逻辑
最近 sequence_check_frames 帧内：
  1. 既有 upright 姿态
  2. 又有 fall 姿态
  3. 当前帧为 fall 姿态
  4. fall_count > upright_count

5.5 Fallback 机制

# SUSPECTED 长期保持但姿态序列不满足
if above_count >= max(sequence_check_frames, alarm_min_frames * 2):
    # 仅跌倒姿态触发 fallback
    if posture in {"crouching", "lying", "unknown"}:
        state = FALLING
重要：sitting/standing 不触发 fallback，防止坐姿误判

核心设计要点
  • ByteTrackLite：两阶段IoU匹配，高分优先+低分补充，提高遮挡鲁棒性
  • 卡尔曼滤波：8维状态向量预测轨迹位置，跳帧时保持跟踪
  • 规则引擎：6条规则覆盖几何、运动、可见性特征
  • 状态机：渐进式状态转换，差异化恢复时间防抖动
  • 跳帧优化：检测稀疏运行，分类器每帧推理，满足边缘延迟
