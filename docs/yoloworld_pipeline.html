YOLO-World 跌倒检测 Pipeline 技术文档

1. 方案概述

核心设计理念
YOLO-World Pipeline 采用开放词汇检测替代传统的「检测→姿态估计→分类器」三阶段流程。 通过训练 YOLO-World 直接识别8种人体姿态类别，省去姿态估计和复杂分类器， 实现端到端单模型推理，显著降低边缘设备部署复杂度。

2. YOLO-World 检测器详解

YOLO-World v2 特性
基于 CLIP 文本编码器的开放词汇检测模型。训练时使用文本嵌入替代固定类别向量， 支持任意类别检测。推理时通过 set_classes() 动态设置检测目标。

开放词汇原理
YOLO-World 将文本嵌入投影到检测头空间，推理时计算视觉特征与文本嵌入的相似度， 选择相似度最高的类别作为检测结果。这使得模型可以检测任意文本描述的物体。

3. 跌倒得分计算

计算策略
YOLO-World Pipeline 的跌倒得分不依赖规则引擎，而是直接基于检测结果。 得分 = 基础分(类别映射) × 检测置信度 + 运动辅助分 + 宽高比辅助分。

3.1 融合得分公式

S_fall = base_score × det_conf + motion_bonus × 0.1 + aspect_bonus
base_score: 类别映射的基础跌倒分（0.0~0.95）
det_conf: YOLO-World 检测置信度
motion_bonus: 垂直快速下降加分（0~0.5）
aspect_bonus: bbox宽高比加分（bbox横向展开时加分）

3.2 运动辅助分

# 基于垂直快速下降给运动加分 (_compute_motion_bonus)
def _compute_motion_bonus(track, history):
    centers = history.get("centers", [])
    if len(centers) >= motion_window_frames + 2:
        recent = centers[-motion_window_frames:]
        vy = recent[-1][1] - recent[-2][1]  # 垂直速度
        if vy > 50:  # px/frame，快速下降
            return min(0.5, vy / 400.0)
    return 0.0
参数默认值说明
motion_window_frames0.5秒 × fps运动计算窗口
vy阈值50 px/frame触发运动加分的最小垂直速度
bonus上限0.5运动加分最大值

3.3 宽高比辅助分

# 倒下的人 bbox 通常 wider than tall
x1, y1, x2, y2 = track.to_tlbr()
w, h = x2 - x1, y2 - y1
aspect = w / max(h, 1.0)

aspect_bonus = 0.0
if aspect > 1.2:  # bbox横向展开
    aspect_bonus = min(0.2, (aspect - 1.2) * 0.3)
设计意图：躺倒姿态的 bbox 通常呈现横向展开（aspect > 1.0），此辅助分弥补检测器偶尔漏检"lying"类别的情况。

4. 融合决策详解

4.1 融合得分公式

S_final = α × S_rule + β × S_cls + γ × S_temporal
YOLO-World Pipeline 中：S_rule = S_cls = S_fall（两者均使用跌倒得分）
默认权重：α=0.3, β=0.5, γ=0.2

5.2 DDP训练修复

已知问题
YOLO-World DDP训练时，非零rank的验证AP接近0，原因是文本嵌入未正确同步到所有GPU。 
# WorldTrainerDDP 修复方案 (src/fall_detection/trainers/yoloworld_trainer.py)
def on_pretrain_routine_end_all_ranks(trainer):
    # 在所有rank上设置类别（不仅是rank 0）
    names = [name.split("/", 1)[0] for name in trainer.test_loader.dataset.data["names"].values()]
    model = unwrap_model(trainer.model)
    model.set_classes(names, cache_clip_model=False)  # 避免DDP内存问题

7. 方案优势总结

简化部署
单模型推理，无需协调多模型，降低边缘部署复杂度
开放词汇
类别可扩展，通过修改文本描述即可新增检测类别
直接姿态识别
直接输出姿态类别，省去姿态估计和分类器推理开销
降低延迟
去除姿态估计环节，推理延迟显著降低
易于调试
检测结果直接关联姿态类别，问题定位更直观
资源友好
适合资源受限的边缘设备（0.5T INT8, 15M DDR）
局限性
  • 姿态检测精度受训练数据质量影响较大
  • 无法提供17个关键点坐标（仅有姿态类别）
  • 对遮挡场景的鲁棒性低于标准Pipeline
  • 类别扩展需重新生成文本嵌入
